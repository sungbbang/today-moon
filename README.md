# 오늘의 달

![썸네일 이미지](./frontend/public/og-img.jpg)

달의 위상과 사용자 위치 기반한 정보를 보여줍니다.

## 만든 이유

달을 보며 힐링하는 개인적인 취미에서 시작했습니다. 날씨나 주변 환경 때문에 실제 달을 관측하기 어려운 상황에서도 오늘의 달이 어떤 모습인지 확인할 수 있도록 만들었습니다.

스터디카페에서 취업 준비를 하던 중, 다른 사람들도 힐링할 수 있길 바라는 마음에 게시판에 QR코드를 붙여 공유했습니다.

## 주요 기능

- 달 위상 이미지 제공

- 현재 위치 기반 월출/월몰 시간 및 달과의 거리, 지역명 표시

- 날짜 직접 선택 및 스와이프 제스처로 날짜 변경

- 소소한 소원 입력 기능 및 카카오톡 공유 기능

## 작업 내용

- 이미지 프리로드 기능 구현

- 달 이미지 좌우 스와이프 제스처를 통한 날짜 이동 구현

- react-day-picker로 날짜 선택 기능 개발

- Geolocation API 활용하여 사용자 위치 기반 정보 표현

- 백엔드에서 Kakao Local API를 연동해 좌표 → 행정구역 변환 API 제작

- TanStack Query로 위치 변환 API 요청 캐싱

- AWS EC2 백엔드 서버 구축

- 카카오톡 공유 기능 구현

## 회고

### 고민한 점

- 달의 모습을 어떻게 보여줘야 하는지

  가장 현실감 있고 괴리감이 적은 이미지 파일을 NASA에서 다운받아 이용하기로 결정

- 달의 정보를 어디까지 표현해야하는지

  오늘 달이 어떤 모습인지와 언제 뜨고, 언제 지고와 같은 핵심 정보에 집중해 간결성 유지

- 날짜 변경 방식을 어떻게 할지

  기존에는 버튼식으로 하루 단위의 변경만 가능했으나, 모바일 환경에서 불편하다는 사용자 피드백을 받고 스와이프 제스처로 변경. react-day-picker를 적용해 사용자가 원하는 날을 직접 선택 가능하게 구현

- 페이지 구조를 어떻게 잡을지

  /moon, /wish와 같이 라우트 분리에 대해 고민했으나 기능이 부족하고 추후 업그레이드 시 도입하는게 적절하다 판단, 라우트 분리없이 개발하기로 결정

- 컴포넌트 변경 시 날짜 유지

  MoonScreen에서 다른 Screen으로 이동하게 되면 보고있던 날짜가 현재 날짜로 리셋되는데 다른 컴포넌트를 탐색하고 다시 돌아왔을 때 날짜는 유지되는게 좋다고 판단, 부모컴포넌트인 App.tsx에 날짜 상태를 관리. 전역상태관리도구를 적용하는 것은 상태관리 구조가 복잡하지 않아 프로젝트에 적합하지 않다고 판단함

### 배운 점

- 무료 호스팅의 콜드스타트와 느린 응답 속도(200~300ms) 문제를 해결하기 위해 AWS EC2로 마이그레이션하면서 Ubuntu 서버 설정, 도메인 연결 등 전반적인 서버 구축 프로세스를 경험. 응답 시간을 50ms 이하로 개선함

- 달에 대한 정보를 어떻게 시각화할지 고민하는 과정이 예상보다 길어졌고 프로젝트를 진행하며 달의 위상, 주기 등 여러 정보를 습득하게 됨. 도메인 이해가 좋은 UX 설계의 필수 요소임을 깨닫게 됨
